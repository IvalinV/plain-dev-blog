# Author Page + Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a public author page (avatar, bio, social link, published-post list) reachable by clicking an author name on any post, plus a class-based Tailwind dark mode with a persisting header toggle.

**Architecture:** Extend the `authors` table with `slug`/`image`/`bio`; reuse the existing `HasUniqueSlug` trait and mirror the `Post`/`PostForm` conventions for the model, Filament form, factory, and Blade views. Dark mode uses a Tailwind v4 `@custom-variant` keyed off a `.dark` class on `<html>`, set pre-paint by an inline script and toggled by a header button.

**Tech Stack:** Laravel 13, PHP 8.4, Filament v5, Tailwind CSS v4, Pest v4.

## Global Constraints

- PHP: always use curly braces for control structures; explicit return types and parameter type hints on all methods; PHPDoc array shapes over inline comments.
- Follow existing conventions in sibling files (check `Post`, `PostForm`, `PostController`, `blog/*.blade.php` before writing).
- Use `php artisan make:` generators with `--no-interaction` for new files.
- Filament v5 form components live under `Filament\Forms\Components\*`; schema under `Filament\Schemas\Schema`.
- Storage: avatars/images use the `public` disk; URLs via `Storage::disk('public')->url(...)`.
- Run `vendor/bin/pint --dirty --format agent` after any PHP change, before committing.
- Tests are Pest feature tests in `tests/Feature`; run with `php artisan test --compact`.
- Frontend/CSS/JS changes require `npm run build` to be reflected in the served app.
- Route by named routes and the `route()` helper.

- [ ] **Step 2: Write the migration body**

Edit the generated migration file. Add `slug` as **nullable first**, backfill any existing rows via the model (so `HasUniqueSlug` generates unique slugs from each author's name), then promote `slug` to `NOT NULL UNIQUE`. This makes the migration safe on databases that already contain authors (a plain `NOT NULL UNIQUE` add with no default fails on 2+ existing rows):

```php
<?php

use App\Models\Author;
use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::table('authors', function (Blueprint $table): void {
            $table->string('slug')->nullable()->after('name');
            $table->string('image')->nullable()->after('social_media');
            $table->text('bio')->nullable()->after('image');
        });

        Author::query()
            ->where(function ($query): void {
                $query->whereNull('slug')->orWhere('slug', '');
            })
            ->get()
            ->each
            ->save();

        Schema::table('authors', function (Blueprint $table): void {
            $table->string('slug')->nullable(false)->change();
            $table->unique('slug');
        });
    }

    public function down(): void
    {
        Schema::table('authors', function (Blueprint $table): void {
            $table->dropUnique(['slug']);
            $table->dropColumn(['slug', 'image', 'bio']);
        });
    }
};
```

Calling `->save()` triggers the `HasUniqueSlug` `saving` hook, which fills a blank slug with a unique value derived from `name`. On a fresh test database (`RefreshDatabase`) the backfill loop runs over zero rows and is a no-op.

```php
<?php

namespace App\Models;

use App\Models\Concerns\HasUniqueSlug;
use Database\Factories\AuthorFactory;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\HasMany;

class Author extends Model
{
    /** @use HasFactory<AuthorFactory> */
    use HasFactory;

    use HasUniqueSlug;

    protected $fillable = [
        'name',
        'email',
        'slug',
        'social_media',
        'image',
        'bio',
    ];

    public function posts(): HasMany
    {
        return $this->hasMany(Post::class);
    }

    protected function sluggableSourceColumn(): string
    {
        return 'name';
    }
}
```

- [ ] **Step 6: Update the factory**

Edit `database/factories/AuthorFactory.php`. Match the `PostFactory` convention: do **not** set an explicit `slug` ŌĆö let `HasUniqueSlug` derive it from the final `name` on save (this also keeps the factory correct when a test overrides `name` without overriding `slug`):

```php
<?php

namespace Database\Factories;

use App\Models\Author;
use Illuminate\Database\Eloquent\Factories\Factory;

/**
 * @extends Factory<Author>
 */
class AuthorFactory extends Factory
{
    public function definition(): array
    {
        return [
            'name' => fake()->name(),
            'email' => fake()->unique()->safeEmail(),
            // slug auto-generated by the model on save
            'social_media' => fake()->optional()->url(),
            'image' => null,
            'bio' => fake()->optional()->sentence(),
        ];
    }
}
```

Note: `HasUniqueSlug` appends `-2`, `-3`, ŌĆ” when two authors resolve to the same base slug, so omitting the explicit slug is collision-safe. The Step 3 model test passes `slug => null` explicitly only to make the intent obvious.

namespace App\Filament\Resources\Authors\Schemas;

use Filament\Forms\Components\FileUpload;
use Filament\Forms\Components\Textarea;
use Filament\Forms\Components\TextInput;
use Filament\Schemas\Schema;

            ->components([

                    ->required()
                    ->maxLength(255),
                TextInput::make('slug')
                    ->helperText('Leave blank to auto-generate from the name.')

                FileUpload::make('image')
                    ->image()
                    ->disk('public')
                    ->directory('authors'),
                Textarea::make('bio')
                    ->rows(3)
                    ->maxLength(500),
            ]);
    }
}
